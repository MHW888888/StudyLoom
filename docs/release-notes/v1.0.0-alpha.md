# StudyLoom / source2study v1.0.0-alpha.0

This is the first public alpha of **StudyLoom**, packaged as the `source2study` Python CLI.

## Positioning

`source2study` is a source-grounded personalized learning pack generator. It helps users turn their own learning or work materials into traceable, personalized, reviewable study/work packs.

It is not a crawler, not a generic video summarizer, and not a platform bypass tool.

## Main Workflow

```text
Source Fidelity -> EvidenceIndex -> Personalized Learning Pack -> Verification -> Export
```

## Supported In This Alpha

- Source Fidelity First intake reports.
- Low-risk local ingestion for Markdown/text, local repos, saved HTML, browser capture JSON, screenshots/OCR sidecars, and transcripts.
- Source and evidence ledgers.
- Personalized learning packs for beginner, exam, developer, creator, teacher, review, and research use cases.
- Markdown, DOCX, and basic PDF export.
- Citation reports and learning-quality reports.
- Deterministic eval suites.
- Restricted MCP/agent tools.
- Curated sample outputs under `examples/outputs/`.

## Demo Commands

```bash
pip install -e .

source2study ingest \
  --workspace ./workspace/demo \
  --source ./examples/demo_sources/notes.md \
  --source ./examples/demo_sources/mini_repo \
  --source ./examples/demo_sources/lecture.vtt

source2study build-index ./workspace/demo
source2study generate ./workspace/demo --mode beginner --output ./workspace/demo/outputs/beginner.md
source2study validate ./workspace/demo --pack ./workspace/demo/outputs/study_pack_beginner_full.json
```

Run evals:

```bash
python evals/run_eval.py --suite standard_demo
python evals/run_eval.py --suite personalized_demo
python evals/run_eval.py --suite degraded_demo
```

Check restricted MCP tools:

```bash
python mcp/server.py --list-tools
```

## Safety Boundaries

This project does not accept requests for:

- cookies or cookie replay
- login sessions or browser profiles
- private headers or tokens
- bulk account-history crawling
- paywall bypass
- DRM bypass
- anti-bot or CAPTCHA bypass
- request-signature reverse engineering
- arbitrary shell execution through agent tools
- arbitrary local file reads through agent tools
- arbitrary URL fetching through agent tools

## Known Limitations

- DOCX/PPTX real structural extraction is planned for v1.1.
- Wiki/MindMap export is planned for v1.2 as an optional source-grounded extension.
- PDF export is basic; Markdown remains canonical.
- OCR currently relies on lightweight sidecar/placeholder paths unless optional engines are added later.
- ASR and keyframe extraction are not default behavior.
- Eval metrics are deterministic rule checks, not human judgment.

## Roadmap

- `v1.1`: DOCX/PPTX real extraction.
- `v1.2`: source-grounded Wiki and MindMap extension.
- `v1.3`: browser extension hardening.
- `v1.4`: optional local OCR/ASR pipeline.
- `v1.5`: template packs.
- `v2.0`: hosted or team workflows.

Experimental platform adapters must remain explicit opt-in and must preserve the same low-risk, user-authorized boundaries.
